    前言

    借着工作中拓展技术栈为目的的项目，接触了后台nestjs框架，看到介绍这个框架是nodejs语言基础上，采用和Spring MVC架构相似的设计思路，用来给用户的服务开发工作提供框架层面的支持。既然是同类型的框架，那么他们的核心原理也是一致的，于是从新了解了框架的核心设计模式-控制反转Ioc。

    技术基础

    1. MVC分层架构
      在了解Ioc之前，我们先得了解MVC的概念。
      MVC架构分别代指Modal、View、Controller，分别用来面向数据管理，视图管理，路由管理等核心层级。
      分层后层级间类的依次调用，构成了整个项目的分层体系架构，这样做的好处是将不同的工作类型解耦，提高代码可扩展程度。
    2. MVC下耦合性问题
      但是面向对象的语言，在分级架构下，会存在另一种常见的问题，对耦合程度的提升影响较大，简述如下：

    baseController.ts

    class baseService {
    constructor () {
    const baseService = new BaseModel();
    }
    baseDao() {}
    }

    baseDao.ts

    class baseController {
    constructor () {
    const newParam = ‘someParam’;
    const baseService = new BaseService(newParam);
    }
    getId() { //… }
    }

    baseService.ts

    class baseDao() {
    constructor (newParam) {
    // ..
    }
    }

    综上，我们可以看到，在dao层这样一个小小的改动，会影响到所有的上层调用方，明显是一种高耦合性的case。如何解决这个问题呢？

    Ioc控制反转的诞生

    我们提出了控制反转Ioc这样的概念，什么叫做控制反转呢？
    就是原先我们调用方来控制被调用方，需要生成一个新对象并缓存等工作。
    现在我们把被调用方控制权交给第三方的控制容器，也就是说调用方不再需要管理被调用方的初始化工作了，这些事情全部由控制容器管理。
    具体是怎么实现的呢？

    baseController.ts

    @Injectable()
    class baseService {
    constructor (private baseDao: BaseDao) {
    // .. 直接使用baseDao
    }
    baseDao() {}
    }

    baseDao.ts

    ```
    @Injectable()
    class baseDao() {
    constructor (newParam) {}
    }

    Ioc的应用

    1. 后台应用
      • Spring MVC

    2. 前台应用
